Artifact Description

For the second enhancement I focused on the recommendation algorithms within the MovieGeek project. The original item‑based neighbourhood recommender re‑computed similarity scores for each request and did not handle new users or sparse data gracefully. I enhanced the algorithm by implementing an in‑memory cache of computed similarities, introducing a cold‑start fallback strategy, and adding defensive programming safeguards.

Key Improvements

  • Caching: Added a dictionary‑based cache keyed on user ID and item ID to store previously computed similarity scores. This reduces the runtime complexity of repeated queries from O(n²) to O(n).
  • Cold‑start fallback: Implemented a fallback recommendation method that returns popular items when there is insufficient user history. This ensures meaningful recommendations for new users.
  • Defensive coding: Added validation checks to prevent division by zero and handle missing ratings. Included comprehensive docstrings describing algorithm parameters and return values.

How This Demonstrates My Skills

This enhancement reflects my ability to analyse algorithmic complexity and optimise data structures for performance. By introducing caching and conditional logic, I demonstrated mastery of time–space trade‑offs and robust algorithm design. Documenting the functions and adding unit tests shows attention to maintainability and correctness.
