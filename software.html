Artifact Description

This artifact is an open‑source web application called MovieGeek implemented with the Django framework. Originally the project contained tightly coupled view logic, hard‑coded configuration values, minimal error handling, and no testing or deployment pipeline. It was created in 2017 as part of the Practical Recommender Systems project. For this enhancement I refactored the code to introduce a service layer, secure configuration, caching, logging, and a continuous integration workflow.

Key Improvements

  • Introduced a dedicated services package to encapsulate business logic, decoupling it from Django views and promoting testability.
  • Secured configuration by loading secret keys and database credentials from environment variables instead of hard‑coding them in source files.
  • Added caching using Django’s caching framework to store expensive computations and reduce page load times.
  • Implemented structured logging and error handling throughout the service layer to improve observability and reliability.
  • Configured a basic continuous integration pipeline that runs unit tests and linting on each commit.

How This Demonstrates My Skills

This enhancement showcases my ability to design maintainable software architectures. Refactoring to a service layer demonstrates an understanding of separation of concerns and modularity. The secure configuration and logging illustrate a security‑conscious mindset. Adding caching and a CI pipeline reflects competence with performance optimisation and industry best practices. Altogether, the enhanced artifact embodies professional software engineering principles.
