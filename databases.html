Artifact Description

The third enhancement addresses the data storage layer of the MovieGeek application. The original database schema relied solely on the default Django indexes and stored configuration settings in the project files. I improved data integrity and query performance by adding composite indexes, creating a recommendation cache table, and refactoring the database configuration to load credentials from environment variables.

Key Improvements

  • Composite index: Added a composite index on the user_id and movie_id fields of the ratings table to speed up lookups for user‑movie pairs.
  • Recommendation cache: Defined a new model for storing precomputed recommendation lists keyed by user. This enables persistent caching across sessions and reduces recomputation.
  • Environment‑based configuration: Removed hard‑coded database credentials from settings.py and replaced them with environment variables, improving security and portability.
  • Validation: Added validators on the rating field to enforce that ratings are between 0 and 5.

How This Demonstrates My Skills

These changes highlight my understanding of relational database design and performance tuning. Creating appropriate indexes and cache tables shows that I can optimise data access patterns. Moving configuration to environment variables demonstrates adherence to security best practices. The improved schema also lays the groundwork for future scaling, as it allows for separate caching and analytics stores.
